# Arsyen Canvas Engine Vision

## Mission
Arsyen Canvas Engine is the narrative experience platform powering portfolios, project presentations, articles, blogs, documentation, knowledge bases, moodboards, idea boards and future long-form creative experiences.

## Not A Website Builder
Canvas is not trying to become Figma, Miro, FigJam, Keynote, PowerPoint or a generic website builder.

## How Canvas Beats Framer
**Framer is a design tool that outputs a website. Arsyen is a narrative engine where meaning is the
source of truth and design is a swappable, AI-directable, multi-surface interpretation.** Framer's
document *is* its pixels — there is no semantic layer beneath them, so the advantages below are ones
its architecture cannot retrofit. We win by going **deep on the narrative niche**, not by matching
Framer's surface area.

### The scorecard (definition of "win" — each row is an epic's exit criterion)
| Axis | Framer | Canvas target | Built by |
|---|---|---|---|
| Instant re-skin (design-language swap) | redesign each variant by hand | one tap re-choreographs every section | E13 |
| Motion / scroll narrative | per-element hand-wiring | declarative, pack-driven, reduced-motion-correct, 60fps | E13 + Interactions |
| AI editing | generates a layout you babysit | edits *meaning* via typed Intent-Patch; reimagine auto-branches | E10 |
| Multi-surface | exports a site | one renderer → web + in-app + Profile, pixel-identical | done (E5/E9) |
| Import / BYOD | pixel-trace, frozen | design-*intent* recovery → editable, restyleable template | E11-v2 |
| Version control | linear history | semantic snapshots / branch / merge; branch-before-AI | E12 |
| Collections | CMS | semantic collections — one design drives N items | E16 |
| Collaboration | multiplayer | CRDT multiplayer on the semantic graph | E17 |
| Taste by default | blank canvas | controlled variation; every output on-brand | E13 + packs |

### Non-goals (the discipline that keeps the moat)
We deliberately do **not** chase: free-form spatial / drag-anywhere layout, generic CMS / arbitrary
data tables, custom-code components, a sprawling component marketplace, pixel-level control. Those
are Framer's territory; pursuing them dilutes the opinionated, narrative-first identity that is the
advantage.

## Philosophy
- Narrative-first
- Vertical scroll experiences
- Content, Structure, Presentation separation
- Design language over styling
- AI-assisted but human-directed
- Creator-native experiences

## Ecosystem Position
Platform = workflows and identity
Research = knowledge production
Generative = AI execution
Canvas = experience creation and publishing
